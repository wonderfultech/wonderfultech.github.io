hadoop streaming left outer join 外连接

有一张用户表：users(user_id,location_id) u1,UT u2,GA u3,CA u4,CA u5,GA u7, 有一张交易表：trans(trans_id,product_id,user_id,quantity,amount) t1,p3,u1,3,330 t2,p1,u2,1,400 t3,p1,u1,3,600 t4,p2,u2,10,1000 t5,p4,u4,9,90 t6,p1,u1,4,120 t7,p4,u1,8,160 t8,p4,u5,2,40 t9,p4,u6,3,50 t10,p3,u7,4,60 现在要求出每个售出的商品对应的唯一用户地址数。 思路分成两个阶段。 算出每个商品对应的地址； 算出每个商品对应的不同地址数，相当于对第一步的去重。 为两个阶段分别设计mapreduce任务。 第一阶段： map.py 将users表中的内容生成：key = user_id 1 value = 1 location_id 将trans表中的内容生成：key = user_id 2 value = 2 product_id 借助mapreduce框架的排序功能。reducer收到的内容中，相同user_id的行，标记为1的行一定排在标记为2的行的前面。 reduce.py 对相同user_id的行，从标记为1的行中取location_id，从排在其后的标记为2的行中取出product_id，并且这些行会共用同一个location_id。从而完成连接运算。 要注意的是：左外连接，按道理左表中的行都要出现的。但是本例只要求出不同地址数，因此在reduce时，之间将location_id不存在的函数删除，因为它们不会影响最终的结果。 这种情况就是例中的两行： t9,p4,u6,3,50 t10,p3,u7,4,60 第二阶段： map2.py 恒等映射 reduce2.py 取出每个产品，并对其对应的不同地址计数。用python中的dict和set数据结构完成计数。 提交脚本：将两个阶段，前后提交mr任务。 最后结果： […]
