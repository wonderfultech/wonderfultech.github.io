leetcode4 寻找两个有序数组的中位数 Median of Two Sorted Arrays

本题等价于从两个有序数组中寻找统一排序后第k个元素。 如果是从单个有序数组中寻找第k个元素，那很容易。 有序数组就考虑用二分方法。但用二分方法容易陷入二分查找算法引起的困境。 二分查找算法，是从有序数组中查找一个元素x的位置。通过x与中间值的大小关系，决定x存在于数组的哪一半。 通过 x值 找 x的位置。 而本题，知道了元素的位置为k，但是不知道这个元素x的值是多少。 通过 x的位置 找 x的值。 这明显有区别。 解法一 数组二分法 设有序数组nums1的区间为[a,b)，中间位置为m1 = a + (b-a)/2。 nums2的区间为[c,d)，中间位置m2 = c + (d-c)/2。 现在要求统一排序后第k个元素。 思路：要从nums1和nums2中逐步缩小第k个元素的所在区间。 首先，要确定一个由nums1和nums2中最小的一批元素组成的区间S。 目的是决定k是否能落在区间S中。 当k 小于 区间S的元素个数时，k落在区间S中。 当k落在区间S中，下一轮查找就会在区间S中进行。这样就能逐步缩小区间S了。 当k没落在区间S中，那么nums1和nums2中不在区间S中的元素就会组成下一轮区间S了。 其次，构造区间S。 当 nums1[m1] <= nums2[m2]时， 可以将nums1[m1]作为区间S的右边界。显然区间nums1[a,m1]中的元素构成区间S的一部分。 另外，区间nums2[c,m2]中可能存在位置pos，使区间nums2[c,pos-1]中的元素都小于等于nums1[m1]，而区间nums2[pos,m2-1]中的元素都大于nums1[m1]。 区间nums2[c,pos-1]中的元素就构成了区间S的另一部分了。 区间S = 区间nums1[a,m1] + nums2[c,pos-1] 。 区间S中的元素都小于等于nums1[m1]，这些元素来自区间nums1[a,m1]和区间nums2[c,m2]。 如何确定位置pos？ 用上界函数upper_bound确定区间nums2[c,m2]中大于nums1[m1]的区间。 函数upper_bound是二分查找算法的一种扩展。 决定k是否落在区间S中 设区间S的元素个数为cnt。 […]
