leetcode124 二叉树中的最大路径和 Binary Tree Maximum Path Sum

解法一 递归法1 实际上基于二叉树后序遍历。 用图直观得展示思路。 图中A节点是一个完整二叉树的一个部分。 考虑求以A为根的子树中的最大路径和。 假设通过递归已经计算出四个值，在图中标记。 myleft_left_max——B节点的左子树中，包含B节点的最大路径和。即，B的左子树中，在所有延申到B节点的路径中，最大路径和值为myleft_left_max。但，要知道，这条路径可能就只有B节点。下同。 其它三个值定义类似。 myleft_right_max——B节点右子树中，包含B节点的最大路径和。 myright_left_max—— C节点的左子树中，包含C节点的最大路径和。 myright_right_max——C节点的右子树中，包含C节点的最大路径和。 那么，以A为根的子树中，最大路径和，出现在三个地方： 1.在B子树中。设值为B_max。 2.在C子树中。设值为C_max。 3.路径可能过A节点。设值为A_max。 情况1和2，已经在递归时完成计算。 现在，只需要计算情况3。 情况3，过A节点。 首先，左子树中，有两种以B为结尾的最大路径和。那么只要从中选出一种即可。 myleft_max = max(myleft_left_max,myleft_right_max); 同样，右子树中，有两种以C为结尾的最大路径和。那么只要从中选出一种即可。 myright_max = max(myright_left_max,myright_right_max); 其次，过A节点路径，不论A节点值是否大于0。都不希望，myleft_max和myright_max小于0。只要， myleft_max和myright_max 中有一个小于0 ，加到A值上，都会减小过A节点路径的和。 因此， myleft_max和myright_max 中 小于0的值，不取。 myleft_max = (myleft_max > 0 ? myleft_max : 0);myright_max = (myright_max > 0 ? myright_max : 0); 最后，以A为根的最大路径和： A_max […]

leetcode45 跳跃游戏 II Jump Game II

我对这道题的解法是存在疑惑的。或许这到题设计之初就是考察贪心算法，用贪心算法来解的。 这道题，我首先想到的是动态规划方法。可惜都超时了。 不同位置的最少跳跃次数存在递推关系。 解法一 动态规划法1（超时） 设d[i]表示从位置0跳到位置i的最少跳跃次数。 这样对任意 j < i，如果j + nums[j] >= i，表示从位置j能跳到位置i，那么就可以用d[j]+1更新d[i]。 这是个O(N^2)的算法。 解法二 动态规划法2（超时） 设d[i]表示从位置0跳到位置i的最少跳跃次数。 从位置i能跳到位置区间S=[i+1,i+nums[i]]。用d[i] + 1更新位置区间S中的d值。 这是个O(N^2)的算法。 解法三 深度优先搜索算法 设d[i]表示从位置0跳到位置i的最少跳跃次数。 实际的思路同解法二，或者叫动态规划算法的递归写法吧。已知d[i]的值，去更新位置i能跳到的位置的d值。 与dfs结合， 在过程中，合理减枝，提高效率。 例如，d[i] + 1 大于等于 d[N – 1]的值时，就不用再计算了。 d[i] + 1 大于等于 d[j]，j是i能跳跃到位置，也不用再计算了。 解法四 广度优先搜索算法 将每个位置当成节点，跳跃关系当成边，就构成了无向图。问题就转化为，无向图中，节点0到节点N-1的最短距离问题。 Hits: 8

leetcode25 K 个一组翻转链表 Reverse Nodes in k-Group
