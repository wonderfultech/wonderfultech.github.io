leetcode124 二叉树中的最大路径和 Binary Tree Maximum Path Sum

Posted on 2019年9月12日2019年12月4日by jasonpeng

leetcode 数据库题目全部题解

Table of Contents

  • 解法一 递归法1
  • 解法二 递归法2

解法一 递归法1

实际上基于二叉树后序遍历。

用图直观得展示思路。

图中A节点是一个完整二叉树的一个部分。

考虑求以A为根的子树中的最大路径和。

假设通过递归已经计算出四个值，在图中标记。

myleft_left_max——B节点的左子树中，包含B节点的最大路径和。即，B的左子树中，在所有延申到B节点的路径中，最大路径和值为myleft_left_max。但，要知道，这条路径可能就只有B节点。下同。

其它三个值定义类似。

myleft_right_max——B节点右子树中，包含B节点的最大路径和。

myright_left_max—— C节点的左子树中，包含C节点的最大路径和。

myright_right_max——C节点的右子树中，包含C节点的最大路径和。

那么，以A为根的子树中，最大路径和，出现在三个地方：

1.在B子树中。设值为B_max。

2.在C子树中。设值为C_max。

3.路径可能过A节点。设值为A_max。

情况1和2，已经在递归时完成计算。

现在，只需要计算情况3。

情况3，过A节点。

首先，左子树中，有两种以B为结尾的最大路径和。那么只要从中选出一种即可。

myleft_max = max(myleft_left_max,myleft_right_max);

同样，右子树中，有两种以C为结尾的最大路径和。那么只要从中选出一种即可。

myright_max = max(myright_left_max,myright_right_max);

其次，过A节点路径，不论A节点值是否大于0。都不希望，myleft_max和myright_max小于0。只要， myleft_max和myright_max 中有一个小于0 ，加到A值上，都会减小过A节点路径的和。

因此， myleft_max和myright_max 中 小于0的值，不取。

myleft_max = (myleft_max > 0 ? myleft_max : 0);
myright_max = (myright_max > 0 ? myright_max : 0);

最后，以A为根的最大路径和：

A_max = max(B_max,C_max,myleft_max + A->val+ myright_max);

还有重要的一步是，A可能是其父节点的左子树或右子树。更新包含A节点的左右子树最大路径和。

A_left_max = myleft_max + A->val;
A_right_max = myright_max + A->val;

综合起来，完整算法：

    typedef long long ll;
    int maxPathSum(TreeNode* root) {
        ll max_v = INT_MIN;
        ll left_max = INT_MIN;
        ll right_max = INT_MIN;
        dfs(root,max_v,left_max,right_max);
        return max_v;
    }
    
    void dfs(TreeNode* rt, ll & max_v, ll & left_max,ll & right_max){
        if(!rt){
            left_max = right_max = INT_MIN;
            return ;
        }
        ll myleft_left_max = INT_MIN;
        ll myleft_right_max = INT_MIN;
        ll myright_left_max = INT_MIN;
        ll myright_right_max = INT_MIN;       
        
        dfs(rt->left,max_v,myleft_left_max,myleft_right_max);
        dfs(rt->right,max_v,myright_left_max,myright_right_max);
        
        ll myleft_max = max(myleft_left_max,myleft_right_max);
        ll myright_max = max(myright_left_max,myright_right_max);
        
        myleft_max = (myleft_max > 0 ? myleft_max : 0);
        myright_max = (myright_max > 0 ? myright_max : 0);
                
        max_v = max(max_v, myleft_max + rt->val + myright_max);
        left_max = myleft_max + rt->val;
        right_max = myright_max + rt->val;            
    }

解法二 递归法2

实际上是对递归法1的改进。也是其它题解中，常出现的解法。

从解法一的讨论中可发现这样的结果。

左子树B中，递归返回的值： myleft_left_max 和 myleft_right_max。

首先，先选出两者之间的最大值。

myleft_max = max(myleft_left_max,myleft_right_max);

其次，如果myleft_max 小于 0，还用不着。

myleft_max = (myleft_max > 0 ? myleft_max : 0);

那么，在对B子树递归时，干嘛要返回 myleft_left_max 和 myleft_right_max。 直接返回，经过上面两步处理后的myleft_max的不就行了吗。

这样简化了代码。

同样方法简化，右子树B中，递归返回的值： myright_left_max 和 myright_right_max。

如此对解法一的代码，进行简化。

    typedef long long ll;
    int maxPathSum(TreeNode* root) {
        ll max_v = INT_MIN;
        ll my_max = INT_MIN;
        dfs(root,max_v,my_max);
        return max_v;
    }
    
    void dfs(TreeNode* rt, ll & max_v, ll & my_max){
        if(!rt){
            my_max = 0;
            return ;
        }
        ll myleft_max = INT_MIN;
        ll myright_max = INT_MIN;
        
        dfs(rt->left,max_v,myleft_max);
        dfs(rt->right,max_v,myright_max);
                
        max_v = max(max_v, myleft_max + rt->val + myright_max);
        my_max = max(myleft_max,myright_max) + rt->val;
        my_max = max((ll)0,  my_max);
    }
