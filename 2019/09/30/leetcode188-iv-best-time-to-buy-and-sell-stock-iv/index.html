解法一 回溯法（超时）

尽管超时，但是好理解，在数据量少时，能work。

回溯法的思路不难，基于这样的考虑。

两种超时的方法

到第i天时，要考虑：

1.还剩余多少次交易机会。

2.在第i天前的某一天有买入股票。

3.已有的利润是多少。

设：k——还剩余多少次交易机会。in——在第in天买入了股票。in<i。当没买入股票时，in=-1。v——已有的利润。

那么在第i天的状态可以用(i,k,in,v)表示。

在第i天能做哪些操作了？

第1种，不操作，不买股票，也不卖股票。然后过度到第i+1天。

第2种，如果之前有买过股票，可卖出股票。in变为-1，利润增加了A[i]-A[in]。

第3种，如果之前没买入股票，可买入股票。in变为i，k减少一次。

那么，整体思路：

ans : 最大利润。
A: A[i]第i天的股票价格。
dfs(A,i,k,in,v){
    if(i == A.length){
        ans = max(ans,v);
    }else{
        //不操作
        dfs(A,i+1,k,in,v);
        if(in != -1){
            //卖出，更新利润
            dfs(A,i+1,k,-1,v+(A[i] - A[in]));
        }else if(k > 0){//还能买入时，买入
            dfs(A,i+1,k-1,i,v);
        }
    }
}

完整代码，超时卡在第206个case上了。

    int maxProfit(int k, vector<int>& prices) {
        int ans=INT_MIN;
        dfs(prices,0,k,-1,0,ans);
        return ans;
    }
    
    void dfs(const vector<int>& A,int cur,int k,int in,int v,int& ans){
        if(cur == A.size()){
            ans = max(ans,v);
        }else{
            //什么也不做
            dfs(A,cur+1,k,in,v,ans);
            if(in != -1){
                //卖出
                dfs(A,cur+1,k,-1,v + (A[cur] - A[in]),ans);
            }else{
                if(k){
                    dfs(A,cur+1,k-1,cur,v,ans);
                }
            }
        }
    }

初步分析，共有N天，每天2种选择。时间复杂度为O(2^N)。因此，只能用在数据量很少的情况。

尽管如此，还是可以进行少量的优化，加速下。 但是，如果不将复杂度降到指数级别以下，很难通过。

第1个优化，当k==0 且 in == -1时，没有必要再递归下去了。

第2个优化，只在价格比买入价格高时，才卖出股票。

修改后的思路：

ans : 最大利润。
A: A[i]第i天的股票价格。
dfs(A,i,k,in,v){
    if(i == A.length 或 k==0 且 in == -1){
        ans = max(ans,v);
    }else{
        //不操作
        dfs(A,i+1,k,in,v);
        if(in != -1){
            //卖出，更新利润
            if(A[i] > A[in])
                dfs(A,i+1,k,-1,v+(A[i] - A[in]));
        }else if(k > 0){//还能买入时，买入
            dfs(A,i+1,k-1,i,v);
        }
    }
}

完整代码，优化后卡在第208个case上了。

    int maxProfit(int k, vector<int>& prices) {
        int ans=INT_MIN;
        dfs(prices,0,k,-1,0,ans);
        return ans;
    }
    
    void dfs(const vector<int>& A,int cur,int k,int in,int v,int& ans){
        if(cur == A.size() || (k == 0 && in == -1)){
            ans = max(ans,v);
        }else{
            //什么也不做
            dfs(A,cur+1,k,in,v,ans);
            if(in != -1){
                //卖出
                if(A[cur] > A[in]) 
                dfs(A,cur+1,k,-1,v + (A[cur] - A[in]),ans);
            }else{
                if(k){
                    //买入
                    dfs(A,cur+1,k-1,cur,v,ans);
                }
            }
        }
    }

因为，这样的小修改，没有改变指数级别的复杂度。

直接从回溯法导出动态规划，可行吗

分析回溯法后，可以发现。在第i天，k，in都相同时，已有的最大利润v，也有可能不同。这说明，在(i,k,in)相同时，v也不一定是最优值。

这是为什么？

因为，虽然在第i天之前，交易的次数是相同的，但是具体在哪一天做了什么操作，在第i天看来是不确定的。这就没法明确前一个状态到达是啥。就没法用递推了。

如果，将每天的两种操作都记录起来，作为状态的一部分，能否可行了？

可是，但就这一点，复杂度就是指数级的了，在数据量大的时候，不可行。

因此，直接从这种回溯法导出动态规划，不可行。

解法二 动态规划

用动态规划来解，不容易想到递推方程。

一旦想通递推方程，股票买卖系列问题的就可以全部解出来了。

这一系列问题有：

买卖股票的最佳时机

买卖股票的最佳时机 II
买卖股票的最佳时机 III
买卖股票的最佳时机 IV

最佳买卖股票时机含冷冻期

买卖股票的最佳时机含手续费

明确状态

定义d[i][j][0 or 1]表示在第i天，进行了j次交易后，持有或不持有1只股票时的最大利润。

在每个状态下，有三种操作：不买也不卖；如果持有股票时，可以卖；如果不持有股票时，可以买。

这样，状态转移就很清晰了。

考虑状态转移方程。

注意：卖出不算1次交易，买入算1次交易。

d[i][j][0]可以从d[i-1][j][0]不买也不卖转过来。也可以由d[i-1][j][1]卖掉一只股票转过来，并加上卖出的价格。要取两种情况的最大值。

可得方程：

d[i][j][0]=max(d[i-1][j][0],d[i-1][j][1] + prices[i])

d[i][j][1]可以从d[i-1][j][1]不买也不卖转过来。也可以由d[i-1][j-1][0]买入一只股票转过来，并减去买入的价格。要取两种情况的最大值。 注意是j-1。

d[i][j][1]=max(d[i-1][j][1],d[i-1][j-1][0] - prices[i])

边界条件

在状态确定后，边界条件非常重要。

当i=0时，

d[0][j][0]=max(d[-1][j][0],d[-1][j][1] + prices[0])

d[0][j][1]=max(d[-1][j][1],d[-1][j-1][0] - prices[0])

那么，两种边界条件：

d[-1][j][0] = 0。还没开始，利润为0.

d[-1][j-1][0] = 0. 利润为0.

d[-1][j][1] = 负无穷。还未开始，不可能持有股票。用负无穷表示不可能持有。

当k=0时，

d[i][0][0]=max(d[i-1][0][0],d[i-1][0][1] + prices[i])

d[i][0][1]=max(d[i-1][0][1],d[i-1][-1][0] - prices[i])

那么，两种边界条件：

d[i-1][0][0] = 0, d[i-1][-1][0] = 0. 没有交易,利润为0.

d[i-1][0][1] = 负无穷, d[i-1][0][1] = 负无穷. 没有交易不可能,拥有股票. 用负无穷表示不可能拥有.

最大利润是:很好理解,显然不能再持有股票,显然要用完天数和交易次数.

d[最后1天][k][0].

那么,对于本题买卖股票的最佳时机 IV,完整的递推方程为.

当i == 0时,
d[i][j][0]=max(d[i-1][j][0],d[i-1][j][1] + prices[i]) = 0
d[i][j][1]=max(d[i-1][j][1],d[i-1][j-1][0] - prices[i]) = -prices[i]
当i == 1时,
d[i][j][0]=max(d[i-1][j][0],d[i-1][j][1] + prices[i])
d[i][j][1]=max(d[i-1][j][1],d[i-1][j-1][0] - prices[i])

这里，要补充两个特殊情况。

第1个，k很大的情况。由于买卖1次要两天，k次要2k天。因此，当天数len < 2k时，再多的k都无意义。所以，对k截断，k=min(k,len/2)。

完整代码为:

    int maxProfit(int k, vector<int>& prices) {
        int ans=INT_MIN;
        const int len = prices.size();
        if(len == 0) return 0;
        k = min(k,len/2);
        vector<vector<vector<int>>> d(len,vector<vector<int>>(k+1,vector<int>(2,0)));
        for(int i=0;i<len;++i){//第i天
            for(int n=0;n<=k;++n){//交易了n次
                if(i == 0){
                    d[i][n][0] = 0;
                    d[i][n][1] = -prices[i];
                }else{
                    d[i][n][0] = max(d[i-1][n][0],d[i-1][n][1] + prices[i]);
                    d[i][n][1] = max(d[i-1][n][1],d[i-1][n-1][0] - prices[i]);
                }
            }
        }
        return d[len-1][k][0];
    }

第2个，天数len很大时，三维数组，超过内存限制。

从数组d的递推关系可知，在第i天，只需要前一天i-1的值即可。

在天数这一维度只需要两维，即:d[2][k+1][2]。

此时，怎么决定第i天选择哪一行了？

对i取模2的余数。

第i天，选择行：s = i & 1。s的值要么是1，要么是0。

当s确定后，第i-1天就是另一行了。

那么，第i-1天，选择行：t = s ^ 1。

因此，完整的代码：

    int maxProfit(int k, vector<int>& prices) {
        const int len = prices.size();
        if(len == 0) return 0;
        k = min(k,len/2);
        const int clen=2;
        vector<vector<vector<int>>> d(clen,vector<vector<int>>(k+1,vector<int>(2,INT_MIN)));
        for(int i=0;i<len;++i){//第i天
            for(int n=0;n<=k;++n){//交易了n次
                if(i == 0){
                    d[i][n][0] = 0;
                    d[i][n][1] = -prices[i];
                }else{
                    const int s = i & 1;
                    const int t = s ^ 1;
                    
                    if(n == 0){
                        d[s][n][0] = 0;
                        d[s][n][1] = -prices[i];
                    }else{
                    
                        d[s][n][0] = max(d[t][n][0],d[t][n][1] + prices[i]);
                        d[s][n][1] = max(d[t][n][1],d[t][n-1][0] - prices[i]);
                    }
                }
            }
        }
        return d[(len - 1) & 1][k][0];
    }

但是，只需要两行的性质不会变。可以将上面的d数组，用变量替换。

替换后的代码：

    int maxProfit(vector<int>& prices) {
        const int len = prices.size();
        if(len == 0) return 0;
        int di10=0,di11=INT_MIN;
        int di20=0,di21=INT_MIN;
        for(int i=0;i<len;++i){//第i天
            di10 = max(di10,di11 + prices[i]);
            di11 = max(di11,- prices[i]);
            di20 = max(di20,di21 + prices[i]);
            di21 = max(di21,di10 - prices[i]);
        }
        return di20;
    }

买卖股票的最佳时机 II的解法(k=无穷大）

题目中要求多次买卖股票。这相当于k不起作用，

在递推方程中，可以将k这一维度去掉。

        for(int i=0;i<len;++i){//第i天
            if(i == 0){
                d[i][0] = 0;
                d[i][1] = -prices[i];
            }else{
                d[i][0] = max(d[i-1][0],d[i-1][1] + prices[i]);
                d[i][1] = max(d[i-1][1],d[i-1][0] - prices[i]);
            }
        }

用di0替换d[i-1][0]，用di1替换d[i-1][1]。注意 d[i-1][0] 和 d[i-1][1] 在两个式子中有交叉引用。

    int maxProfit(vector<int>& prices) {
        const int len = prices.size();
        if(len == 0) return 0;
        int di0 = 0, di1 = INT_MIN;
        for(int i=0;i<len;++i){//第i天
            int a = di0;
            di0 = max(di0,di1 + prices[i]);
            di1 = max(di1,a - prices[i]);
        }
        return di0;
    }

买卖股票的最佳时机含手续费的解法(k=无穷大+手续费)

题目中允许无限次的完整交易。k也不再起作用。

但要收取手续费，这相当于在卖出股票时，扣除手续费。

在上面的解法中，扣除费用即可。

    int maxProfit(vector<int>& prices, int fee) {
        const int len = prices.size();
        if(len == 0) return 0;
        const int clen=2;
        vector<vector<int>> d(clen,vector<int>(2,INT_MIN));
        for(int i=0;i<len;++i){//第i天
            if(i == 0){
                d[i][0] = 0;
                d[i][1] = -prices[i] - fee;
            }else{
                d[i][0] = max(d[i-1][0],d[i-1][1] + prices[i]);
                d[i][1] = max(d[i-1][1],d[i-1][0] - prices[i] - fee);
            }
        }
        return d[(len - 1) & 1][0];        
    }

用di0替换d[i-1][0]，用di1替换d[i-1][1]。注意 d[i-1][0] 和 d[i-1][1] 在两个式子中有交叉引用。

    int maxProfit(vector<int>& prices, int fee) {
        const int len = prices.size();
        if(len == 0) return 0;
        int di0 = 0, di1 = INT_MIN;
        for(int i=0;i<len;++i){//第i天
            int a = di0;
            di0 = max(di0,di1 + prices[i]);
            di1 = max(di1,a - prices[i] - fee);
        }
        return di0;        
    }

最佳买卖股票时机含冷冻期的解法(k=无穷大+1天冷冻期)

匀速多次交易。但是卖出后，至少隔开1天后才能再买入股票。

这里，k也相当于无效。

要强调的是，递推方程：

            d[i][1] = max(d[i-1][1],d[i-2][0] - prices[i]);

而且边界条件d[-2][0]值为0.

    int maxProfit(vector<int>& prices) {
        const int len = prices.size();
        if(len == 0) return 0;
        const int clen=3;
        vector<vector<int>> d(clen,vector<int>(2,INT_MIN));
        for(int i=0;i<len;++i){//第i天
            if(i == 0){
                d[i][0] = 0;
                d[i][1] = -prices[i];
            }else if(i == 1){
                d[i][0] = max(d[i-1][0],d[i-1][1] + prices[i]);
                //这个地方尤为重要d[-2][0]的值为0
                d[i][1] = max(d[i-1][1],-prices[i]);
            }else{// >= 2
                const int s = i % clen;
                const int t = (i - 1) % clen;
                const int u = (i - 2) % clen;
                d[s][0] = max(d[t][0],d[t][1] + prices[i]);
                d[s][1] = max(d[t][1],d[u][0] - prices[i]);
            }
        }
        return d[(len - 1) % 3][0];        
    }

继续对代码进行简化。

用di0替换d[i-1][0]，用di1替换d[i-1][1]。注意 d[i-1][0] 和 d[i-1][1] 在两个式子中有交叉引用。 用di2代替d[i-2][0]，其值是更新前的di0。这好理解，因为只有三行。

int maxProfit(vector<int>& prices) {
    const int len = prices.size();
    if(len == 0) return 0;
    int di0 = 0, di1 = INT_MIN;
    int di2 = 0;
    for(int i=0;i<len;++i){//第i天
        int a = di0;
        di0 = max(di0,di1 + prices[i]);
        di1 = max(di1,di2 - prices[i]);
        di2 = a;
    }
    return di0;
}
