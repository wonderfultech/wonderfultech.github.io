leetcode214 最短回文串 Shortest Palindrome

Posted on 2019年9月10日2019年12月4日by jasonpeng

leetcode 数据库题目全部题解

找到以第一个字符开头的最长回文串t。将剩余的串反转后添加到串s前面即可。

解法一 kmp解法

先介绍下kmp算法。

用字符串，aacecaaa，作为模式串P。来匹配其它文本串S。

用一个张图，来解释模式串P，用kmp匹配的工作原理。

图上有三个部分，上部是文本串S，中部是模式串及其右移，下部是状态机。

图中，P与S在位置7处发生不匹配。

朴素查找法，会将P右移1位，在从P的第一个字符开始匹配。图中，右移1位，显示这种情况。

在图中，当右移5位时，P与S潜在要比较的字符数有6个，最少。因为此时P的前缀aa与P的后缀aaa，相同部分aa，最长。

这样，就意味着，当P位置7发生不匹配时，应该将P右移5位，让位置2处的字符与S串的位置7对齐，并开始比较。并且P位置2之前的字符一定是匹配。

状态机中的节点数字——已经匹配了i个字符。向右的箭头——每匹配一个字符，进入下一个状态。向左的曲线箭头——失配边或失配函数，失配时要回到哪个状态。

在上面的情况中，用曲线箭头，从位置7指向位置2。就表示，当位置7匹配失败时，状态切换到位置2，P右移，从位置2开始继续匹配。

用同样的方法，计算其它位置的失配函数，显示在图中状态机上。

那么，kmp的关键就是求出每个状态的失配函数。

怎么找？

设P失配的位置i，字串P1=P[0,i-1]，要找出P1的最长前缀串Pre，使得Pre也是P1的后缀串。设当Pre=P1[0,j]时，位置i失配的值为位置j。

其实就是模式串P，自己与自己匹配。

用这个方法就能得出图中其它状态的失配函数。

失配函数的计算算法。

记住，失配时沿着失配边往回走。

void fail(const string& P,vector<int>& f){
    f[0] = 0;
    f[1] = 0;
    for(int i = 1;i < P.size(); ++i){
        int j = f[i];
        while( j && P[i] ! = P[j] ) j = f[j];
        f[i + 1] = P[i] == P[j] ? j+1 : 0;
    }
}

回到这道题本身。将s的逆串r，拼接到s后面，即ss = s + "#"+r。

本题的关键就是求出ss的一个最长前缀串Pre，且Pre是ss的后缀串。

从kmp的失败函数计算过程中，可得出Pre = [0,f[ss.length] - 1]。

本题的解为：r[0, r.length - f[ss.length] - 1] + s

完整算法：

    string shortestPalindrome(string s) {
        const int N = s.size();
        if(N <= 1) return s;
        string r(s);
        reverse(r.begin(),r.end());
        string ss(s + "#" + r);
        vector<int> f(ss.size() + 1,0);
        fail(ss,f);
        return r.substr(0,r.size() - f[ss.size()]) + s;
    }

解法二 Manacher 马拉车算法

原先用于在线性时间内解决最长回文子串问题。算法难在理解，代码却很简单。

Manacher算法的核心是用已经得到的最长回文串，推算新位置的最长回文串。

例如，求aacecaaa的最长回文串。

借助下面的表格计算最长回文串。

第一行——下标。

第二行——字符填充后的串S。至于为什么要填充，后面再讲解。

第三行——P[i]表示位置i处，以S[i]为中心的最长回文串的长度 / 2 + 1。又叫回文半径。先了解下，后面再解释。

现在，依次计算每个位置i的P值。 计算过程有些长。

01234567891011121314151617
S$#a#a#c#e#c#a#a#a#
P112321218121234321

S[0] 左边是边界，P[0] = 1。

S[1]，S[0] != S[2]，P[1] = 1。

S[2]，S[1] == S[3]，P[2] = 2。

S[3]，S[1] == S[5]，P[3] = 3。

在S[4]处，发现S[4]包含在S[3]的回文串S[1,5]中。回文串是对称的。S[4]关于S[3]的对称位置为S[2]。

那么，P[4]的可能值之一是P[2]。

要知道，S[4]的回文串可能更长，超过S[1,5]右侧的范围。但是S[1,5]右侧的字符是否是S[4]的回文串的一部分还未知。因此，能确定的，P[4]的可能值之一是2，就是位置4和5的长度，5 - 4 + 1 = 2。

这样，目前为止，能断定P[4]的最小值为min(P[2], 5 - 4 + 1)=2。

但这不是P[4]的最终值。因为P[4]可能在S[1,5]之外的值，继续匹配，那么P[4]的值会增加。

S[0]和S[6]不匹配，P[4]的值为2。

至此，S[4]的回文串S[3,5]，还在S[3]的回文串S[1,5]中。

用求P[4]的方法求出P[5] = 1。

S[6]其超出了S[3]的回文串S[1,5]。只能用左右逐个字符匹配的方法得出P[6]=2。

S[7]在S[6]的回文串S[5,7]中，用求P[4]的方法求出P[7]=1。

S[8]超过了S[5,7]的范围。 用左右逐个字符匹配的方法得出P[8]=8。

那么，S[8]的回文串是S[1,15]。其覆盖了之后的S[9,15]的位置

用求P[4]的方法，逐个求P[9,15]的值。

P[9]=min(P[7],15 - 9 + 1) = min(1,7) = 1。

S[8]和S[10]不匹配，因此P[9]=1。

P[10]=min(P[6],15 - 10 + 1) = min(2,6) = 2。

S[8]和S[12]不匹配，因此P[10]=2。

P[11]=min(P[5],15 - 11 + 1) = min(1,5) = 1。

S[10]和S[12]不匹配，因此P[11]=1。

P[12]=min(P[4],15 - 12 + 1) = min(2,4) = 2。

S[10]和S[14]不匹配，因此P[12]=2。

P[13]=min(P[3],15 - 13 + 1) = min(3,3) = 3。

S[10]和S[16]不匹配，因此P[13]=3。

P[14]=min(P[2],15 - 14 + 1) = min(2,2) = 2。

S[16]和S[12]匹配， S[17]和S[11]匹配，因此P[14]=2 + 2 = 4。

S[14]的回文串S[11,17]右侧，超过了 S[8]的回文串S[1,15]。覆盖了位置15，16和17。

P[15]=min(P[13],17 - 15 + 1)= min(3,3) = 3。

位置18越界，因此P[15]=3。

P[16]=min(P[12],17 - 16 + 1)= min(2,2) = 2。

位置18越界，因此P[16]=2。

P[17]=1。

上面的计算过程，已经应用了 Manacher算法 的核心逻辑。

将其整理如下：

  1. 计算过程是从左向右。递推计算P[i]，且P[i] >= 1。
  2. 并且当S[i]不是#和$时，P[i] >= 2。也就是说，新串中最长回文串，一定包含旧串中的字符，而且两端必是#。
  3. 设S[k]的回文串S[a,b]（k - a = b - k）是已经算出的回文串中，b最大的回文串。
  4. 如果S[i]落在了，S[a,b]（k - a = b - k）的覆盖范围内。要利用S[k]的回文串的对称性。此时，P[i]的最小长度为min(P[2*k - i], b - i + 1)。类似例子中演示的那样，P[4],P[9,15]等。
  5. 如果S[i]在S[a,b]的右边，还有字符匹配，则P[i]的长度要继续增加。如P[14]。
  6. 每计算一个P[i]，其回文串S[c,d]。如果d>b则，则S[i]的S[c,d]是下一个位置计算中，右边界最大的回文串。

填充#的原因，#要不同于原串中的字符：原始串的两头和字符之间插入#，是为了让新串长度为奇数，便于算法计算，不用考虑偶数长度的回文串。原串长度n，新串为n+n+1=2n+1。#字符的个数一定 等于 旧串长度 + 1。

开头插入$的原因：将新串的下标都改为从1开始，旧串的字符都在新串偶数位置。便于计算。

现在,已经找到了,新串中的最长回文子串的位置max_i和回文半径P[max_i]。如何计算旧串中的回文子串的位置old_i和长度old_len？

首先，通过填充$#后，旧串的字符都在新串偶数位置。设 新串 偶数位置q，其在旧串的下标为(q-2)/2。

要明确，新串的最长回文串，是位置S[max_i]的回文串S[max_i - P[max_i] + 1, max_i + P[max_i] - 1]，总长度是2*P[max_i] - 1。

这个回文串中，第一个字符一定是#，位置为 max_i - P[max_i] + 1 。

第二个字符才是旧串中的字符，位置为 max_i - P[max_i] + 2。

那么，

old_i = (max_i - P[max_i] + 2 - 2) / 2 = (max_i - P[max_i]) / 2。

其次，由于填充#，#的个数一定等于旧串长度+1。

有，old_len + old_len + 1 = 2*P[max_i] - 1 。

old_len = P[max_i] - 1。

那么 Manacher算法 的代码就容易理解了：

// s是旧串
        string t("$#");
        for(int i=0;i<s.size();++i){
            t+=s[i];
            t+='#';
        }
        //P[i]
        vector<int> p(t.size()+5,1);
        //新串中最长回文子串的起始位置
        int max_i=0;
        //center，right分别是右边界最大的回文子串的位置i和右边界+1。
        int center=0,right=0;
        //迭代计算P[i]
        for(int i=1;i<t.size();++i){
            if(i<right){
                p[i]=min(p[2*center-i],right-i);
            }
            //至少有p[i]长度，但是也有可能还可以继续扩展。
            while(i+p[i] < t.size() && i-p[i] >= 1 && t[i+p[i]] == t[i-p[i]]) ++p[i];
            if(p[i] > p[max_i]){
                max_i=i;
            }
            //更新右边界+1
            if(i+p[i] > right){
                right=i+p[i];
                center=i;
            }
        }

回到本题中用Manacher算法求解。只需记录，以旧串第一个字符为始的最长回文子串。即，记录(max_i-p[max_i])/2 == 0时， P[max_i] - 1 最大的子串。

完整算法代码为：

    string shortestPalindrome(string s) {
        const int N = s.size();
        if(N < 2) return s;
        string t("$#");
        for(int i=0;i<s.size();++i){
            t+=s[i];
            t+='#';
        }
        vector<int> p(t.size()+5,1);
        int max_i=0;
        int center=0,right=0;
        //以旧串第一个字符为始的最长回文子串的长度
        int len = 1;
        for(int i=1;i<t.size();++i){
            if(i<right){
                p[i]=min(p[2*center-i],right-i);
            }
            //至少有p[i]长度，但是也有可能还可以继续扩展。
            while(i+p[i]<t.size() && i-p[i]>=1 && t[i+p[i]] == t[i-p[i]]) ++p[i];
            if(p[i]>p[max_i]){
                max_i=i;
            }
            if(i+p[i]>right){
                right=i+p[i];
                center=i;
            }
            //以旧串第一个字符为始的最长回文子串
            if((max_i-p[max_i])/2 == 0){
                len = max(len,p[max_i] - 1);
            }
        }
        if(len == N) return s;
        string sub = s.substr(len);
        reverse(sub.begin(),sub.end());
        return sub+s;
    }

解法三 暴力法（超时）

得到反转串r。通过串s和r，求出最长回文串。

设回文串长度为len。那么有s[0,len) == r[N - len]。

len从N减到1，逐个判断，求出最长回文串。

    string shortestPalindrome(string s) {
        const int N = s.size();
        if(N <= 1) return s;
        string r(s);
        reverse(r.begin(),r.end());
        for(int len=N;len >= 0;--len){
            if(s.substr(0,len) == r.substr(N - len)){
                return r.substr(0,N - len)+s;
            }
        }
        return string();
    }

解法四 递归法

换一种方法找出以第一个字符开始的最长回文串。

用指针i从头开始，指针j从尾部向前。

当 s[j] == s[i] 时，i++。

当j遍历完s时，s[0,i]部分包含潜在的以第一个字符开始的最长回文串，s[i+1,N)部分就是要反转的字符串。

对s[0,i]部分再递归求解以第一个字符开始的最长回文串。

    string shortestPalindrome(string s) {
        const int N = s.size();
        if(N <= 1) return s;
        int i = 0;
        for(int j = N - 1;j >= 0;--j){
            if(s[i] == s[j]) ++i;
        }
        if(i == N) return s;
        string r = s.substr(i);
        reverse(r.begin(),r.end());
        return r+shortestPalindrome(s.substr(0,i))+s.substr(i);
    }
