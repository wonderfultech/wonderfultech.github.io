leetcode315 计算右侧小于当前元素的个数 Count of Smaller Numbers After Self

Posted on 2019年10月13日2019年12月4日by jasonpeng

leetcode 数据库题目全部题解

Table of Contents

  • 解法一 归并排序法
  • 解法二 树状数组解法

解法一 归并排序法

归并排序将序列先分成两个相同长度的子序列，分别排序后，再归并两个有序的子序列。

在归并排序的过程中，顺便完成个数统计。

关键在于归并操作。

例如：[-1, 5, -5, -5, -4, -2, -3, 5, 2, -5] N=10

分成： [-1, 5, -5, -5, -4 ] [ -2, -3, 5, 2, -5]

假设已经将子序列排好序后，并且每个元素的右侧比其小的个数都求出来了。

左边 的结果

0	1	2	3	4
---------------------------------
-5	-5	-4	-1	5
0	0	0	3	3

右边的结果

5	6	7	8	9
---------------------------------
-5	-3	-2	2	5
0	1	2	1	2

我们看最后一步归并。

A[0] = -5 <= A[5] = -5 取结果T[0] = A[0]

A[1] =-5 <= A[5] = -5 取结果T[1] = A[1]

A[2] =-4 > A[5] = -5 取结果T[2] = A[5]

A[2] = -4 < A[6] = -3 取结果T[3] = A[2]，但是A[2]右边比A[2]小元素的个数要加上 第二个子序列中 A[6]前 的元素个数1. COUNTS[3] = 0 + 1 =1。

为什么？

因为取A[2]表示在第二个子序列中A[6]之前的元素都小于A[2]，因此一定要加上。

A[3] = -1 > A[6] = -3 取结果T[4] = A[6]

A[3] = -1 > A[7] = -2 取结果T[5] = A[7]

A[3] = -1 < A[8] = 2 取结果T[6] = A[3] COUNTS[6] = 3 + 3 =6。第二个3表示 在第二个子序列中A[8]之前的小于A[3] 的个数。

A[4] = 5 > A[8] = 2 取结果T[7] = A[8]

A[4] = 5 <= A[9] = 5 取结果T[8] = A[4] COUNTS[4] = 3 + 4 =7。第二个4表示 在第二个子序列中A[9]之前的小于A[4] 的个数。

取结果T[9] = A[9] 。

从上面的可看出，在归并操作中，当取第一个子序列中的元素a时，为其增加第二个子序列中比a小的元素个数。

但是要强调一个case。

例如:[2,1,2]

我们知道其结果是[1,0,0].

如果在排序的过程中第一个2和第二个2交换了位置，那么最终结果是[0,0,1]。这显然是错误的。

这要求排序算法必须是稳定的。而归并排序是稳定的。

因此，本算法不会出现上述的问题。

整体思路：

A[x,...,y)序列归并排序;
merge_sort(A,COUNTS,T,x,y){
    m = x + (y-x)/2;
    merge_sort(A,C,x,m);
    merge_sort(A,C,m,y);
    a = x, b = m;
    p = x;
    while(a < m || b < y){
        if(b == y || a < m && A[a] <= A[b]){
            COUNTS[a] += b - m;
            T[p++] = A[a++];
        }else T[p++] = A[b++];
    }
    A[x,...,y) = T[x,...,y);
}

但是，实际上为了将下标与元素和个数绑定起来，方便计数。

在数字数组A上又加了层索引数组W。归并排序过程中，元素顺序的调整，只会修改索引数组W。

增加了索引数组W后的完整代码：

    vector<int> countSmaller(vector<int>& nums) {
        const int N = nums.size();
        vector<int> W(N),T(N);
        vector<int> C(N,0);
        for(int i=0;i<nums.size();++i){
            W[i] = i;
        }
        merge(nums,W,T,C,0,N);
        return C;
    }
    
    void merge(const vector<int>& A,
              vector<int>& W,
              vector<int>& T,
              vector<int>& C,
              int x,int y){
        if(y - x <= 1) return;
        int m = x+(y-x)/2;
        merge(A,W,T,C,x,m);
        merge(A,W,T,C,m,y);
        int a=x,b=m;
        int p=x;
        while(a < m || b < y){
            if(b == y || a < m && A[W[a]] <= A[W[b]]){
                C[W[a]] += b-m;
                T[p++] = W[a++];
            }else{
                
                T[p++] = W[b++];
            }
        }
        for(int i = x;i<y;++i){
            W[i] = T[i];
        }
    }

解法二 树状数组解法

树状数组解决动态前缀和问题。

数组A[1,...,N]，对A[i]进行修改，同时还能以最快的速度求位置j(j>=i)处的前缀和。

用树状数组解决本题的思路：

第一.求出每个数字在排序后数组中的序号。注意排序要是稳定的。

例如：[5,2,6,1]

排序后：[1,2,5,6]。那么，[5,2,6,1]序号数组为[3,2,4,1]。

例如:[2,1,2]

排序后:[1,2,2]。排序要是稳定的。

第二.从右向左遍历序号数组,取序号R[i]，再在树状数组C[R[i]]处加上1。取数组C位置R[i]前的前缀和，就是位置i处的结果。

怎么理解？

可以这么理解。在上例中，6的序号是4，即6排在第4位。其前面有3个数字。

但是在原数组[5,2,6,1]中，6右边只出现了1，还有两个没有出现。因此，不能算入个数中。

接着看上例， 序号数组为[3,2,4,1]

R[3] = 1，在C中1下标处填1，C[R[3]]=C[1] = 1。C=[0,1,0,0,0]。

在C中1下标 前的前缀和为0.因此A[3]=1的右侧小于1的元素个数为0.

R[2]=4， 在C中4下标处填1，C[R[2]]=C[4] = 1。C=[0,1,0,0,1]。

在C中4下标 前的前缀和为1.因此A[2]=6的右侧小于6的元素个数为1.

R[1]=2， 在C中2下标处填1，C[R[1]]=C[2] = 1。C=[0,1,1,0,1]。

在C中2下标 前的前缀和为1.因此A[1]=2的右侧小于2的元素个数为1.

R[0]=3， 在C中3下标处填1，C[R[0]]=C[3] = 1。C=[0,1,1,1,1]。

在C中3下标 前的前缀和为2.因此A[0]=5的右侧小于5的元素个数为2.

结果是[2,1,1,0]。

树状数组本身不难理解。

每个水平的长条表示其下方覆盖的位置上A中元素的和。 LOWBIT表示长条覆盖的元素的个数。

例如：C[4]=A[1]+A[2]+A[3]+A[4]

要算A[11]的前缀和，在上图中，沿着黑线求和。结果是：C[11]+C[10]+C[8]

LOWBIT(x) = x 的 二进制 表达式 中最 右边 的 1 所 对应 的 值。

LOWBIT(6) = 0110b 中 0010b的值2.

LOWBIT(0) = 0

LOWBIT(11) = 1011b 中 0001b的值1.

现在更新了A[3]处的值，从图上可看到，位置3，被C[3],C[4]和C[8]覆盖了。因此，只需修改C[3],C[4],C[8]的值。

可以看出树状数组的操作，时间复杂度是O(logN)。

树状数组的代码：

    //在位置i处加上1   
    void put(vector<int>& C,int i){
        while(i < C.size()){
            C[i] += 1;
            i += lowbit(i);
        }
    }
    //取位置i的前缀和。
    int sum(vector<int>& C,int i){
        int r = 0;
        while(i > 0){
            r += C[i];
            i -= lowbit(i);
        }
        return r;
    }

    int lowbit(int i){
        return i & -i;
    }

整体思路：

同解法一，依然用索引数组间接排序。

索引数组W；
序号数组Rank;
结果数组R;

用W间接对A排序;
算出Rank；
for(i in [N-1,...,0]){
    //将排名添加到树状数组
    put(C,Rank[i]);
    //取位置Rank[i]处的前缀和
    R[i] = sum(C,Rank[i])-1;
}

完整代码：

    struct cmp{
        vector<int>& W;
        const vector<int>& A;
        cmp(vector<int>& _W,const vector<int>& _A):W(_W),A(_A){}
        bool operator()(const int i,const int j)const{
            return A[i] < A[j] || A[i] == A[j]  && i < j;
        }
    };

    vector<int> countSmaller(vector<int>& nums) {
        const int N = nums.size();
        vector<int> rank(N);
        vector<int> W(N);
        vector<int> C(N+5,0);
        vector<int> R(N);
        for(int i=0;i<N;++i){
            W[i] = i;
        }
        sort(W.begin(),W.end(),cmp(W,nums));
        for(int i=0;i<N;++i){
            rank[W[i]] = i+1;
            //printf("%d:%d ",nums[W[i]],i+1);
        }
        for(int i=N-1;i>=0;--i){
            put(C,rank[i]);
            R[i] = sum(C,rank[i]) - 1;
        }
        return R;
    }
