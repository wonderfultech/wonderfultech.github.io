leetcode32 最长有效括号 Longest Valid Parentheses

Posted on 2019年10月11日2019年12月4日by jasonpeng

leetcode 数据库题目全部题解

Table of Contents

  • 解法一 区间动态规划（超时）
  • 解法二 基于栈的一维度动态规划
  • 解法三 一维动态规划

解法一 区间动态规划（超时）

算法在第218个case上超时了。 在动态规划算法中，这是最容易想到的算法了。

思路：设子串长度为len，len为偶数。判断s串的子串s[i,...,i+len-1]是否有效，如果有效则得出len长度的子串。i向右移动到i+1,i+2,...,等，逐一判断子串的是否有效。

当len=2，4，6，8，...，随着len的增加，子串长度增加。依次判断长度为len的子串是否有效。

设d[i][i+len-1]表示子串s[i,...,i+len-1]是否有效。有效性由两个条件决定。

第一，当s[i] == ( 且 s[i+len-1] == )时，有效性取决于s[i+1,...,i+len-2]的有效性。

d[i][i+len-1] = d[i+1][i+len-2] ;

例如："((()))"

s[0] == ( 且 s[5] == )，d[0][5] 取决于d[1][4]。

第二，当k = [i+2,i+4,...,j-2]时，如果s[i,...,k-1]和s[k,...,j]都有效时，s[i,...,j]也有效.

for(k in [i+2,i+4,...,j-2]){
    d[i][j] = d[i][j] || d[i][k-1] && d[k][j];
}

例如："((()))"

d[0][5] 取决于 d[0][1]和d[2][5] ; d[0][3]和d[4][5] 。

完整思路：

for(len in [2,4,6,...,s.length]){
    for(i in [0,...,N-len]){
        j = i+len-1;
        if(len == 2){
            d[i][j]= s[i] ==( && s[j] == );
        }else{
            if( s[i] ==( && s[j] == )   ){
                d[i][j] = d[i][j] || d[i+1][j-1];
            }
            for(k in [i+2,i+4,...,j]){
                d[i][j] = d[i][j] || d[i][k-1] && d[k+1][j];
            }
        }
        if(d[i][j]有效) 记录最长的len;
    }
}

完整代码：

    int longestValidParentheses(string s) {
        const int N = s.size();
        vector<vector<int>> d(N+1,vector<int>(N+1,0));
        int ans = 0;
        for(int len=2;len<=N;len+=2){
            for(int i=0;i+len-1<N;++i){
                const int j = i+len-1;
                if(len == 2){
                    d[i][j] = (s[i] == '(' && s[j] == ')');
                }else{
                    for(int k=i+2;k < j && !d[i][j];k+=2){
                        d[i][j] = d[i][j] || d[i][k-1] && d[k][j];
                    }
                    if(!d[i][j] && s[i] == '(' && s[j] == ')'){
                        d[i][j] = d[i][j] || d[i+1][j-1];
                    }
                    
                }
                if(d[i][j]) ans = max(ans,len);
            }
        }
        return ans;
    }

解法二 基于栈的一维度动态规划

在解法一的基础上将起点固定在0，那么d[0][j]就不能再表示有效性了，而直接表示s[0,...,j]之间以s[j]为结尾的最长有效串的长度。直接用d[j]表示这个长度。

思路：从左到右遍历，s[i]=(时入栈S。

当s[i]=)时，如果栈S为空，说明没有(与之匹配，pass掉。

当栈不为空，说明栈顶与之匹配。此时串s[S.top(),...,i]是有效串。

有效串的长度=i-S.top()+1。

另外，栈顶位置S.top()的前一个位置S.top()-1的值d[S.top()-1]也要加到d[i]上。

d[i] = i-S.top()+1 + d[S.top()-1]

匹配完后，栈顶出栈。

思路很清晰，代码很简洁。

    int longestValidParentheses(string s) {
        const int N = s.size();
        int ans = 0;
        vector<int> d(N+1,0);
        stack<int> S;
        for(int i=0;i<N;++i){
            if(s[i] == '('){
                S.push(i);
            }else if(S.size()){
                int pre = S.top() - 1;
                int len = (pre >= 0 ? d[pre] : 0)+(i - S.top() + 1);
                ans = max(ans,len);
                d[i] = len;
                S.pop();
            }
        }
        return ans;
    }

解法三 一维动态规划

解法一超时说明O(N^2)复杂度的算法是不行的。

在解法一的基础上将起点固定在0，那么d[0][j]就不能再表示有效性了，而直接表示s[0,...,j]之间以s[j]为结尾的最长有效串的长度。直接用d[j]表示这个长度。

此时唯一能用的就是j之前已经算出的长度。

考虑中间位置k ，k = [j-2,j-4,j-6,...]。

那么d[j] = max(d[k] + len);

如果s[j-k+1,...,j]有效则len = j-k，否则位置k对d[j]的长度没有贡献。

但是，会引入 s[j-k+1,...,j] 有效性的判断。这个思路也是可行的，请看解法二。

如果将k限制在j-2。串s[j-1,j]的有效性判断就很直观。

如果s[j-1] == ( 且 s[j] == )时， s[j-1,j] 有效。

d[j] = d[j-2]+2。

例如:s=")()"

d[2]=2+d[0]=2

如果s[j-1] == ) 且 s[j] == )时， s[j-1,j] 无效。

但此时d[j-1]已经算出，d[j-1]决定了子串s[j-d[j-1],...,j-1]是有效的。

如果此时s[j-d[j-1]-1]==(，说明子串 s[j-d[j-1]-1,...,j-1]是有效的。

d[j] = d[ j-d[j-1]-2] + 2 + d[j-1]

例如：s="))(())"

d[5] = d[1] + d[4] +2 =4

思路很清晰，完整的代码：

    int longestValidParentheses(string s) {
        const int N = s.size();
        vector<int> d(N+1,0);
        int ans = 0;
        for(int i=1;i<N;++i){
            if(s[i-1] == '(' && s[i] == ')'){
                d[i] = 2 + (i-2 >=0 ? d[i-2] : 0);
            }else if(s[i-1] == ')' && s[i] == ')' && i - 1- d[i-1] >= 0 && s[i - 1- d[i-1]] == '('){
                d[i] = d[i-1] + 2 + (i - 2 -d[i-1]>=0 ? d[i - 2 -d[i-1]] : 0);
            }
            ans = max(ans,d[i]);
        }

        return ans;
    }
