leetcode41 缺失的第一个正数 First Missing Positive

Posted on 2019年10月6日2019年12月4日by jasonpeng

leetcode 数据库题目全部题解

Table of Contents

  • 解法一 将nums[i]放置在i
  • 解法二 将nums[i]放置在nums[i]-1

解法一 将nums[i]放置在i

解法思路：

当碰到nums[i]不是i时，将nums[i]调整到其应该的位置，即nums[i]。

也就是交换位置i和位置nums[i]上的值。之后，如果位置i上的值nums[i]依然不满足nums[i]!=i。则继续进行交换。直到，nums[i]==i或nums[i]的值不在[0,nums.size()-1]的范围内。

这些不在范围内的和重复的正数值，则变成负数，不再考虑。

结果是从最终序列中，找出第一个nums[i] != i的值i。

例如：[1,2,0]

nums[0] != 0 交换nums[0]和nums[1]得 [2,1,0]。

nums[0] != 0 交换nums[0]和nums[2]得 [0,1,2]。

例如：[3,4,-1,1]

nums[0] != 0 交换nums[0]和nums[3]得 [1,4,-1,3]。

nums[0] != 0 交换nums[0]和nums[1]得 [4,1,-1,3]。

由于4超出范围，则讲其变为负数，再考虑。 [-4,1,-1,3]

例如：[7,8,9,11,12]

所有值，全部超出范围，全部变为负数。 [-7,-8,-9,-11,-12]

但是，有三种特殊情况。

第一种，序列没有元素或只有一个元素。

例如：[]

例如： [1],[-1],[2]

第二种，每个位置都满足nums[i]==i。这时，结果是最后元素+1或序列长度。

例如:[0,1,2]

例如:[0,1,2,3,4]

第三种，1，2，3，4，...的一种排列，不缺少一个数。这样的序列，通过上面的方法，最终形成的序列是最大值在位置0，其它位置依次为1,2,3,4,5....。

这时，结果是最大元素+2或序列长度+1.

例如：[1,2,3,4,5] 交换后序列为 [-5,1,2,3,4]

例如：[1,2,3,5,4,6,7] 交换后序列为[-7,1,2,3,4,5,6]

完整代码：

    int firstMissingPositive(vector<int>& nums) {
        const int len = nums.size();
        if(len == 0) return 1;
        if(len == 1){
            if(nums[0]<=0) return 1;
            if(nums[0] == 1) return 2;
            else return 1;
        }

        for(int i = 0;i < nums.size();++i){
            while(nums[i] != i && nums[i] < nums.size() && nums[i]>0){
                int next = nums[i];
                if(nums[i] == nums[next]){
                    nums[i] = -nums[i];
                    break;
                }
                swap(nums[i],nums[next]);
            }
            if(nums[i] >= (int)nums.size()){
                nums[i] = -nums[i];
            }
        }
        int i = 1;
        for(i = 1;i < nums.size();++i){
            if(nums[i] != i){
                break;
            }
        }
        if(i ==  nums.size()) {
            i=nums.back()+1;
            if(-(nums.back() + 1) == nums[0]){
                i = max(i,nums.back() + 2);
            }
        }
        return max(1,i);
    }

从上面的思路中可发现，i是一直增加的，i不会变小。因此，将哪些超过范围或重复的正数值变成负数，没有必要。

解法二 将nums[i]放置在nums[i]-1

思路基本同解法一，逐个判断每个位置是否满足nums[i]==nums[nums[i]-1]。但是避免nums[i]是最小值时的溢出，改为判断nums[i] == i+1。

相比于解法一，这样放置有什么好处了？

依然考虑解法一中的三个特殊情况。

第一种，序列没有元素或只有一个元素。

例如:[]

例如：[1],[-1],[2]

第二种，每个位置都满足nums[i]==i。这时，结果是第一个不满足nums[i] == i+1 的i+1。

例如:[0,1,2] 交换后的序列[1,2,0]

例如:[0,1,2,3,4] 交换后的序列[1,2,3,4,0]

第三种，1，2，3，4，...的一种排列，不缺少一个数。这样的序列，通过上面的方法，最终形成的序列是最大值在位置0，其它位置依次为1,2,3,4,5....。 这时，结果是最大元素+1或序列长度+1.

例如：[1,2,3,4,5] 交换后序列为 [1,2,3,4,5]

例如：[1,2,3,5,4,6,7] 交换后序列为[1,2,3,4,5,6,7]

这样，将第一种情况中的[]和第三种情况的结果归为一类，其结果是序列长度+1.

第二种情况和普通的情况是一样的，这简化了结果判断，也简化了代码。

完整代码：

    int firstMissingPositive(vector<int>& nums) {
        for(int i = 0;i < nums.size();++i){
            while(nums[i] != i+1 && nums[i] <= nums.size() && nums[i] > 0){
                int next = nums[i] - 1;
                if(nums[i] == nums[next]){
                    break;
                }
                swap(nums[i],nums[next]);
            }
        }
        for(int i = 0;i < nums.size();++i){
            if(nums[i] != i+1){
                return i+1;
            }
        }
        return nums.size()+1;
    }
