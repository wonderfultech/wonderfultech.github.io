解法一 欧拉回路算法 本题的第一个难点在于理解题意。 题目要求出包含所有可能密码的最短密码串。 总共有k^n个密码，每个密码n个字符。直接将这些密码拼起来，共有(k^n)n个字符。这样的串不是最短的。 利用密码之间公共部分，重复的部分保留一份。既能包含所有的密码，又能使串最短。 密码之间的公共部分最长是n-1个字符。 构造有向图 采用题目提示中的思路。 构建一个k^(n-1)个节点的有向图Grid。每个节点有k个有条边。 在Grid找出一条欧拉回路。回路的边构成密码。欧拉回路和欧拉道路。 密码的每一位是 k 位序列 0, 1, …, k-1 中的一个 。 首先，k^(n-1)个节点是指由n-1个密码字符组成的串。 其二，k个有向边。从一个n-1个密码字符组成的串s中，截取尾部n-2个字符子串。尾部拼接上 0, 1, …, k-1 中的一个 ，组成下一个 n-1个密码字符组成的串t。 从s到t就形成了一条有向边。 其三，每个节点。有k个出边，也有k个入边。总度数是2k，是偶数。因此满足欧拉回路存在的充要条件——所有顶点的度数是偶数。 欧拉道路存在的充要条件——要么0个，要么2个顶点的度数是奇数。当然图本身要是连通的。 其四，在欧拉回路寻找过程中，构建有向图。不用实现构造有向图。这个后面再讲。 例如：n=3，k=2 用上述算法形成的有向图。 节点00. 尾部0，拼上0，形成00。边:00->00. 尾部0，拼上1，形成01。边:00->01. 求出欧拉道路 欧拉回路本质上，从起点s开始，每条边恰好走一遍，再回到起点s。 顶点可以重复出入，但是边只能走一次。这样的方式与dfs算法类似。 dfs算法是对顶点只访问一次。 这里改造dfs算法，将原来的顶点去重，改为边去重。 去重数组vis[u][v]。 其次，dfs算法对顶点的遍历是前序遍历。 在这里，对边遍历改为后序遍历。如果边是前序遍历，处理的路径序列，就构不成欧拉回路。 基于欧拉道路d构造密码串。 d[i]中u和v有n-2个字符的公共的串。 同时d[i]和d[i-1]之间有n-1个字符的公共的串。 拼接这些串时，将公共的部分仅保留一份。 这样就形成了最短密码串. 例如：n=3，k=2 用上述dfs算法得出的欧拉道路： 思路： 代码： Hits: 19
