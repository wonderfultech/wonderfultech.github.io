leetcode668 乘法表中第k小的数 Kth Smallest Number in Multiplication Table

Posted on 2019年12月11日2019年12月11日by jasonpeng

leetcode 数据库题目全部题解

解法一 二分查找

应用矩阵中的二分查找算法，类似的题目有，leetcode74 搜索二维矩阵。

本题中，关键点是对什么进行二分？

第一种是对坐标(x,y)进行二分。二分后(x/2,y/2)，计算出此处的值为v = (x/2 * y/2)。

然后在数组中，查找所有小于等于v的数字的个数。

第二种是直接对数字(m*n)二分。

第一次二分，数字为v = (m*n)/2. 然后在数组中，查找所有小于等于v的数字的个数。

第二次二分，数字为v = (m*n)/4. 然后在数组中，查找所有小于等于v的数字的个数。

....

可以看出，第二种情况包含了第一种情况。都要落实到具体的数字。

第二个关键点是怎么在矩阵中， 查找所有小于等于v的数字的个数。

例如：m=12，n=12，k=17

第一次二分，v=12*12/2=72.

找出小于等于v=72的数字个数。

乘法表中，

第1行中的数字都是1的倍数。因此第1行中小于等于v=72的个数是min(12,72/1)=12。

第2行中的数字都是2的倍数。因此第2行中小于等于v=72的个数是min(12,72/2)=12。

第3，4，5，6行，小于等于v=72的个数是12.

第7行中的数字都是7的倍数。因此第7行中小于等于v=72的个数是min(12,72/7)=10。

同样，第8，9，10，11，12行， 小于等于v=72的个数是9，8，7，7，6.

如此， 小于等于v=72的数字个数 为119.

119 > k=17。说明v=72太大了。还要缩小。

再次二分v=(12*12)/4=36.

用上面的方法求出 小于等于v=36的数字个数.

a=1,b=m*n;
while(a < b){
    mid=(a+b)/2;
    cnt=0;
    for(i in [1,m]){
        cnt += min(n,mid/i);
    }
    if(cnt < k) a=mid;
    else b=mid;
}
b是结果;

代码：

    int findKthNumber(int m, int n, int k) {
        int a=1,b=m*n;
        while(a < b){
            int mid=a+(b-a)/2;
            if(mid == a) break;
            int cnt=0;
            for(int i=1;i<=m;++i){
                int q=mid/i;
                if(q==0) break;
                cnt += min(q,n);
                if(cnt >= k) break;
            }
            if(cnt < k){
                a=mid;
            }else if(cnt >= k){
                b=mid;
            }
        }
        return b;
    }
