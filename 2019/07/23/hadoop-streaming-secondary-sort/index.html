Posted on 2019年7月23日2019年8月30日by jasonpeng

leetcode 数据库题目全部题解

hadoop streaming 能让脚本或可执行程序作为map/reduce的任务。

命令格式: genericOptions 一定要放在 streamingOptions之前。

hadoop jar hadoop-streaming.jar [genericOptions] [streamingOptions]

举一个二次排序的例子

输入文件：sampe_input.txt 格式：年,月,日,温度。

2000,12,04,10
2000,11,01,20
2000,12,02,-20
2000,11,07,30
2000,11,24,-40

2012,12,23,60
2012,12,24,70
2012,12,25,10
2013,01,22,80
2013,01,23,90
2013,01,24,70
2013,01,20,-10

想要得出输出：年-月 温度列表，从小到大。

2000-11 -40,20,30
2000-12 -20,10
2012-12 -20,10,30,60,70
2013-01 -10,70,80,90

设计MR任务

map.py

#!/usr/bin/env python
#coding=utf-8
import sys
for line in sys.stdin:
    line=line.strip()
#    print(line)
    #year,month,day,temperature
    seg=line.split(',')
    #重新组成:year,month,temperature,temperature,day
    groupKey=seg[0]+','+seg[1]+','+seg[3]
    groupValue=seg[3]+','+seg[2]
    print('%s,%s'%(groupKey,groupValue))

reduce.py

#!/usr/bin/env python
#coding=utf-8
import sys
kkey=''
vvalue=''
for line in sys.stdin:
    line=line.strip()
    #print(line)
    parts=line.split('\t')
    #parts[0] key
    #parts[1] value
    keys=parts[0].split(',')
    nkey=keys[0]+'-'+keys[1]
    if nkey != kkey:
        if kkey!='':
            print('%s\t%s' % (kkey,vvalue))
            kkey=''
            vvalue=''
    kkey=nkey
    if vvalue=='':
        vvalue=keys[2]
    else:
        vvalue+=','+keys[2]
print('%s\t%s' % (kkey,vvalue))

run.sh 提交脚本

#!/bin/bash
#hadoop 2.7.6
JAR=${HADOOP_HOME}/share/hadoop/tools/lib/hadoop-streaming-2.7.6.jar
INPUT=/chap01/input/sample_input.txt
OUTPUT=output
MAPPER=map.py
REDUCER=reduce.py
hadoop fs -rm -r  $OUTPUT
hadoop jar $JAR \
-D stream.map.input.ignoreKey=true \
-D stream.map.output.field.separator=, \
-D stream.num.map.output.key.fields=3 \
-D mapreduce.map.output.key.field.separator=, \
-D mapreduce.partition.keypartitioner.options=-k1,2 \
-D mapreduce.job.output.key.comparator.class=org.apache.hadoop.mapreduce.lib.partition.KeyFieldBasedComparator \
-D mapreduce.partition.keycomparator.options=-k1,3 \
-input $INPUT \
-output $OUTPUT \
-mapper $MAPPER \
-reducer $REDUCER \
-file $MAPPER \
-file $REDUCER \
-partitioner org.apache.hadoop.mapred.lib.KeyFieldBasedPartitioner

脚本中重要选项的说明：

-D stream.map.output.field.separator=, —— 指定","为map输出的分隔符。

-D stream.num.map.output.key.fields=3 —— 第三个","之前的部分作为map输出的key，也就是组合键部分，之后的部分作为map输出的value。

-D mapreduce.map.output.key.field.separator=, —— 指定map输出的key中的分隔符为","。

注意：不建议将 stream.map.output.field.separator 和 mapreduce.map.output.key.field.separator 设置为'\t'，特别是要二次排序，并且数字值排序时，会导致数字值排序失效为文本串排序。

-D mapreduce.partition.keypartitioner.options=-k1,2 —— 指定map的输出的key中的自然键部分为第1和第2个字段，并按第1和第2个字段进行划分。

-D mapreduce.job.output.key.comparator.class=org.apache.hadoop.mapreduce.lib.partition.KeyFieldBasedComparator —— 指定map输出的key的排序类，即组合键的排序类。

-D mapreduce.partition.keycomparator.options=-k1,3 —— 指定map输出的key的排序字段，为第1到第3字段。其实就是组合键部分与 stream.num.map.output.key.fields 的值要对应。

详解 mapreduce.partition.keycomparator.options 和mapreduce.partition.keypartitioner.option

mapreduce.partition.keycomparator.options，其值传给 org.apache.hadoop.mapreduce.lib.partition.KeyFieldBasedComparator，用来提取组合键排序选项。

而mapreduce.partition.keypartitioner.option，其值传给org.apache.hadoop.mapred.lib.KeyFieldBasedPartitioner。

这两个选项的参数格式完全相同。

hadoop-3.1.2的源代码中，参数格式说明：

/**
* This is used in {@link KeyFieldBasedComparator} &
* {@link KeyFieldBasedPartitioner}. Defines all the methods
* for parsing key specifications. The key specification is of the form:
* -k pos1[,pos2], where pos is of the form f[.c][opts], where f is the number
* of the field to use, and c is the number of the first character from the
* beginning of the field. Fields and character posns are numbered starting
* with 1; a character position of zero in pos2 indicates the field's last
* character. If '.c' is omitted from pos1, it defaults to 1 (the beginning
* of the field); if omitted from pos2, it defaults to 0 (the end of the
* field). opts are ordering options (supported options are 'nr').
*/

翻译：

用在 KeyFieldBasedComparator 和 KeyFieldBasedPartitioner 中。定义用于解析关键字规范的所有方法。关键字规范的形式：-k pos1,[pos2]，其中pos的格式形如f[.c][opts]，其中f是要用的字段序号，c是从字段起点开始的第一个字符的序号。字段和字符位置的序号都从1开始。pos2中字符位置为0表示字段的最后一个字符。如果从pos1中删除'.c'，它的默认值为1（字段的开始）。如果将其从pos2中删除，它的默认值为0（字段的结束）。opts是排序选项（支持的选项有'nr'）。

两个选项：

-n, (Sort numerically) 按数字排序

-r, (Reverse the result of comparison) 逆序

参数的解析在 KeyFieldHelper中完成。每个 '-k pos1,[pos2]' 都对应一个字段，被解析为一组关键字的参数：

  1. beginFieldIdx 选项1，2，对应到pos1。选项1对应'.'之前的值，2对应到其后的值。
  2. beginChar
  3. endFieldIdx 选项3，4，对应到pos2。选项3对应'.'之前的值，4对应到其后的值。 另外，要注意，参数格式中'[,pos2]'是可选的，可能不存在。当不存在时， endFieldIdx 和 endChar 都为0，表示到字段的结束。
  4. endChar
  5. numeric
  6. reverse

举几个例子

假设参数用于排序。

-k2,2n

按第2个字段的数字值排序。

-k2,2

按第2个字段的文本值排序。

-k3.4,3n

按第3个字段中从第4个字符开始的部位的数字值排序。

-k3.2,3.3n -k4,4nr

按第3个字段中从第2个字符到第3个字符的部位的数字值排序，同时按第4个字段的数字值逆序。

-k2.4,2.4n

按第2个字段中第4个字符的数字值排序。

现在要求：

<字符串1>,<浮点数> 为组合键。 <字符串2> 为值。

要按 <字符串1> 分区。但是要按组合键排序。并且要求，<字符串1> 按字符串排序， <浮点数> 按数字值排序。

如何配置streaming选项？

分析：

由于第2个','之前的部分为组合键部分。有选项：

-D stream.map.output.field.separator=,
-D stream.num.map.output.key.fields=2

并且组合键内部以','分割，有选项：

-D mapreduce.map.output.key.field.separator=,

又要求按第一个字段分区。因此有选项：

-D mapreduce.partition.keypartitioner.options='-k1,1'

同时要求，按第1个字段文本排序，第2个字段数字排序。有选项：

-D mapreduce.job.output.key.comparator.class=org.apache.hadoop.mapreduce.lib.partition.KeyFieldBasedComparator
-D mapreduce.partition.keycomparator.options='-k1,1 -k2,2n'
