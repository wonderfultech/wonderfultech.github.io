Redis skiplist 跳跃表

Posted on 2019年7月21日2019年8月30日by jasonpeng

leetcode 数据库题目全部题解

In redis, skiplist is introduced as an internal data structure constructing zset.

The skiplist is an extension of sorted linked list. It allows average O(nlogn) search, insertion and deletion complexity. In paper -"Skip Lists: A Probabilistic Alternative to Balanced Trees", it has been analyzed carefully. We first recall some key points about it.

The nodes in skiplist have forward pointers to the nodes ahead of them. These forward pointers are parts that are different from general linkedlist that has only one forward pointer.

The number of pointers in a node is called the level of the node. The levels of nodes are choosen randomly. With the help of pointers, we can skip over intermediate nodes.

Search Algorithm

We search for a node by traversing forward pointers that do not overshoot the node. When no more progress can be made at the current level, the search moves down to the next level. When we can not make progress at level 1, we should be immediately in front of the node if it exists in the list.

Insertion and deletion algorithm

Search is the basic operation in both algorithms. Search right place for insertion and where is the node for deletion.

The curve in the image is the trail of search. Except the search, the rest of both algorithm are obvious.

The level of new node is selected randomly. We decide a constant p first, which indicates the ratio of the notes with level i pointers also have level i+1 pointers. We have a algorithm for selecting level randomly like this.

Then, a problem emerges. How to decide the level at that do we start a search? The paper suggests that we would start a search at the level L where we expect 1/p nodes. So, L=log(1/p,n). Also, we should choose MaxLevel=L(N), where N is an upper bound on the number of elements in a skiplist. We will not give the proof of complexity in paper here.

We get a summary of skiplist in paper above. But skiplist in redis has some different points from the one in paper.

Differences:

  • list can hold duplicate value in it;
  • comparison between key includes score and object;
  • node has a backforward pointer.
  • node has a field -- span supporting for rank.
  • list has a dump node.
