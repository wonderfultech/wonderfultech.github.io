Redis object 对象

Posted on 2019年7月21日2019年8月30日by jasonpeng

leetcode 数据库题目全部题解

用对象表示数据库中的键和值。

对象类型：数据库中键只能是字符串对象，值可以是全部支持的对象。支持的对象有：

  • 字符串对象
  • 列表对象
  • 哈希表对象、
  • 集合对象
  • 有序集合对象

编码类型：对象使用的编码，即这个对象的底层实现用了什么数据结构。支持的数据结构有：

一个对象可以有多种底层的数据结构支持。 对象类型与编码类型有如下的对应关系。

对象的定义：字段意义清晰。

typedef struct redisObject {
unsigned type:4;
unsigned encoding:4;
unsigned lru:REDIS_LRU_BITS; 
int refcount;
void *ptr;
} robj;

refcount--引用计数是对象复用的关键点。lru--用于内存回收操作。

  1. 引用计数：refcount
    1. 创建时，refcount=1；
    2. 被新用户使用时，refcount++；
    3. 不再被新用户使用时，refcount--；
    4. refcount==0时，对象被释放；
  2. 共享对象：多为保存整数值的字符串对象。
    1. 将指针指向已有对象；
    2. 对象refcount++；
  3. lru字段:对象最后一次被命令程序访问的时间。OBJECT IDLETIME不会修改lru字段。用于内存回收。

对象支持的操作函数：

createObject(int type,void * ptr):创建一个robj对象；type类型；ptr实际存储数据的内存；
	zmalloc分配内存robj，初始化各个字段；
	注意：不分配ptr执行的内存；
createRawStringObject(char* ptr,size_t len):创建字符串对象，ptr中指向的是sds结构。
createEmbeddedStringObject(char* ptr,size_t len):为字符串ptr创建obj和sds串。obj和sds串连在同一块内存上。
createStringObject(char* ptr,size_t len):
	<=39:createEmbeddedStringObject
	其它：createRawStringObject
createStringObjectFromLongLong(long long value):为整数创建一个字符串对象
	若value符合REDIS共享整数的范围，增加共享对象引用计数，返回共享对象；
	否则创建字符串对象，保存整数；
createStringObjectFromLongDouble(long double value):为long double创建一个字符串对象
	转换位17位小数的字符串；
	移除尾部的0；
	createStringObject创建；
dupStringObject(robj* o):复制一个字符串对象，有相同的编码类型；
	根据o->encoding的类型，分别调用createRawStringObject，createEmbeddedStringObject,
	createObject。

	创建dict；

freeStringObject:释放字符串对象

freeSetObject：释放集合对象；要分dict和intset；
freeZsetObject:释放有序集合；要分skiplist和ziplist；
freeHashObject:释放哈希对象；要分dict和ziplist；
incrRefCount:增加引用计数；
decrRefCount:减少引用计数；
	如果减少后为0，根据对象类型，释放对象；
	REDIS_STRING->freeStringObject:释放字符串对象

	REDIS_ZSET->freeZsetObject:释放有序集合；要分skiplist和ziplist；
