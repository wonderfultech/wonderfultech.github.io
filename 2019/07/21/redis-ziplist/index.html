Redis ziplist 压缩列表

Posted on 2019年7月21日2019年8月30日by jasonpeng

leetcode 数据库题目全部题解

Ziplist 是为了尽可能地节约内存而设计的特殊编码双端链表。

Ziplist 只储存字符串值和整数值，其中，整数值被保存为实际的整数，而不是字符数组。

Ziplist 允许在列表的两端进行 O(1) 复杂度的 push 和 pop 操作。 但是，因为这些操作都需要对整个 ziplist 进行内存重分配，所以实际的复杂度和 ziplist 占用的内存大小有关。

节点结构：

previous_entry_length指示前一个节点的字节长度，便于从后向前遍历。逆序遍历列表有些麻烦。

编码说明：编码在ziplist中非常重要。

级联更新：节点长度超过254字节，其下一个节点的previous_entry_length用一个字节不能表示。这样会扩展下一个节点的previous_entry_length，以至于下一个节点变长。

会造成级联更新的函数：push,insert,delete,deleteRange.

Ziplist支持的函数：

New：新建立一个空白的ziplist。
Resize：调整ziplist的resize。
__cascadeUpdate:当插入新节点时，设置下一个节点的prevlen空间不够大时，要级联扩展后续节点的空间。
__delete:连续删除n个节点。更新删除区域的下一个节点的prevlen，前移剩余的节点，更新尾部偏移，更新链表长度。
__insert：在位置p处，插入长度为slen的字符串s。
	计算新节点的长度；
	Resize扩展长度；
	后移插入点之后的节点；
	__cascadeUpdate之后节点的header
	放置新节点；
Push：在头部或尾部插入长度为slen的字符串。
	调用__insert完成。
Index：定位索引index处的节点。
	index为正，从前往后；
		跳过节点长度，到下一个节点；
	index为负从后向前。
		跳过prevlen长度，到前一个节点；
Next:返回p节点的下一个节点。
	跳过节点长度，到下一个节点
Prev：返回p节点的前一个节点。
	跳过prevlen长度，到前一个节点。
Get：取出节点p的值。
	如果是字符串，取出字符串首地址和长度；
	如果是整数，解码并取出整数；
Insert：在位置p处插入新节点。
	调用__insert完成。
Delete：删除节点p。
	调用__delete完成。
DeleteRange：范围删除从index开始的n个节点；
	调用Index确定删除开始位置p；
	调用__delete删除，区域；
Compare：比较位置p处节点与参数的值。
	如果是字符串，比较字符串；
	如果是整数，解码，比较整数。
Find：查找与参数值相等的节点，并且每次跳过skip个节点。
	如果是字符串，比较字符串；
	如果是整数，解码，比较整数。
