Thomas Wang’32 & 64 mix function

Posted on 2019年7月20日2019年8月30日by jasonpeng

leetcode 数据库题目全部题解

In redis, Thomas Wang'32 bit mix function is used as a kind of hash function. Actually it is a mix function. It has only reversible operations in it. So, there is a one-to-one mapping between the input and the output. We will prove that later. Beside reversibility, the avalanche of it is unobvious. It not easy to verify that. Even, the author also did not provide a clear proof of the avalanche in original posts. http://web.archive.org/web/20071223173210/http:/www.concentric.net/~Ttwang/tech/inthash.htm http://web.archive.org/web/20060507103516/http:/www.cris.com/~Ttwang/tech/inthash.htm

So, we only talk about the reversiblity here. Also, we discuss 64bit version of mix function in the rest.

This is the code of Thomas Wang'32 bit mix function in redis.

unsigned int dictIntHashFunction(unsigned int key)
{
    key += ~(key << 15);
    key ^= (key >> 10);
    key += (key << 3);
    key ^= (key >> 6);
    key += ~(key << 11);
    key ^= (key >> 16);
    return key;
}

The extended gcd algorithm lays an critical foundation in proving the resersibility.

It is well known that the basic gcd algorithm is Euclid's algorithm.

int gcd(int a,int b){
	Return b==0 ? a : gcd(b,a%b);
}

We can change it a little bit to solve the problem like this.

The Problem One: Find a integer pair (x,y) that satisfys the equation ax+by=gcd(a,b). Here a and b are also integers.

A extended gcd algorithm has been promoted here. Here, d is the gcd(a,b).

void gcd(int a,int b,int & d,int & x,int & y){
	if(!b) { d=a; x=1;y=0;}
	else {
		gcd(b,a%b,d,y,x);
		y-=x*(a/b);
	}
}

Let's go further a little bit. A conclusion can be made here.

The Conclusion: a,b,c are integers. g=gcd(a,b). The equation ax+by=g has a solution (x0,y0). When c is the multiple of g, the equation ax+by=c has the solution (x0*c/g,y0*c/g). Or, if c is not the multiple of g, there is no integer solution.

Go ahead! Another important problem blocks the road.

The Problem Two: a, b , n are positive integers. Find x that satisfy the equation ax = b (mod n) .

We get a equivalent expression of this problem. ax-b must be the positive multiple of n. That is ax-b=ny. Then, ax-ny=b is the final equal formula. This equation can be solved with the help of the extended gcd algorithm!

Make a equivalent modification on Thomas Wang'32 bit mix fuctions.

unsigned int reverse_dictIntHashFunction(unsigned int key)
{
    uint32_t k6=key;
    uint32_t k5=k6^(k6>>16);
    uint32_t k4=4290770943 * ~k5;
    uint32_t k3=k4^(k4>>6)^(k4>>12)^(k4>>18)^(k4>>24)^(k4>>30);
    uint32_t k2=954437177 * k3;
    uint32_t k1=k2^(k2>>10)^(k2>>20)^(k2>>30);
    uint32_t k0=3221192703  * ~k1;
    return k0;
}

Until now, we have verified reversibility of Thomas Wang'32 bit mix function and got the reverse function. In the same way, we also can get the reverse function of its 64bit version.

uint64_t thomasWang64MixFunction(uint64_t key)
{
  key = (~key) + (key << 21); // key = (key << 21) - key - 1;
  key = key ^ (key >> 24);
  key = (key + (key << 3)) + (key << 8); // key * 265
  key = key ^ (key >> 14);
  key = (key + (key << 2)) + (key << 4); // key * 21
  key = key ^ (key >> 28);
  key = key + (key << 31);
  return key;
}
uint64_t inverse_hash(uint64_t key) {
	uint64_t k;
	//reverse of key = key + (key << 31)
	k = key-(key<<31);
	key = key-(k<<31);
	//reverse of key = key ^ (key >> 28)
	k = key^key>>28;
	key = key^k>>28;
	//reverse of key *= 21
	key *= 14933078535860113213u;
	//reverse of key = key ^ (key >> 14)
	k = key^key>>14;
	k = key^k>>14;
	k = key^k>>14;
	key = key^k>>14;
	//reverse of key *= 265
	key *= 15244667743933553977u;
	//reverse of key = key ^ (key >> 24)
	k = key^key>>24;
	key = key^k>>24;
	//reverse of key = (~key) + (key << 21)
	k = ~key;
	k = ~(key-(k<<21));
	k = ~(key-(k<<21));
	key = ~(key-(k<<21));
	return key;
}
