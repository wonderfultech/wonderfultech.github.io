hadoop streaming left outer join 外连接

Posted on 2019年7月24日2019年8月30日by jasonpeng

leetcode 数据库题目全部题解

有一张用户表：users(user_id,location_id)

u1,UT
u2,GA
u3,CA
u4,CA
u5,GA
u7,

有一张交易表：trans(trans_id,product_id,user_id,quantity,amount)

t1,p3,u1,3,330
t2,p1,u2,1,400
t3,p1,u1,3,600
t4,p2,u2,10,1000
t5,p4,u4,9,90
t6,p1,u1,4,120
t7,p4,u1,8,160
t8,p4,u5,2,40
t9,p4,u6,3,50
t10,p3,u7,4,60

现在要求出每个售出的商品对应的唯一用户地址数。

思路分成两个阶段。

  1. 算出每个商品对应的地址；
  2. 算出每个商品对应的不同地址数，相当于对第一步的去重。

为两个阶段分别设计mapreduce任务。

第一阶段：

map.py

将users表中的内容生成：key = user_id 1 value = 1 location_id

将trans表中的内容生成：key = user_id 2 value = 2 product_id

借助mapreduce框架的排序功能。reducer收到的内容中，相同user_id的行，标记为1的行一定排在标记为2的行的前面。

!/usr/bin/python
#coding=utf-8
import sys
for line in sys.stdin:
    line = line.strip()
    #可能是来自users或trans，要区分
    segs=line.split(',')
    #print(segs)
    if len(segs)==2:
        #来自users
        print('%s\t1\t1\t%s'%(segs[0],segs[1]))
    elif len(segs)==5:
        #来自trans
        print('%s\t2\t2\t%s'%(segs[2],segs[1]))

reduce.py

对相同user_id的行，从标记为1的行中取location_id，从排在其后的标记为2的行中取出product_id，并且这些行会共用同一个location_id。从而完成连接运算。

要注意的是：左外连接，按道理左表中的行都要出现的。但是本例只要求出不同地址数，因此在reduce时，之间将location_id不存在的函数删除，因为它们不会影响最终的结果。

这种情况就是例中的两行：

t9,p4,u6,3,50
t10,p3,u7,4,60

#!/usr/bin/python
#coding=utf-8
import sys
uid=''
loc=''
for line in sys.stdin:
    line=line.strip()
    #但排在前面的肯定是uid 1 1 locationid。
    #还可能是uid 2 2 productid
    segs=line.split('\t')
    #print(line,segs)
    if len(segs)!=4:
        #过滤掉不规格的行
        continue
    if segs[2]=='1':
        uid=segs[0]
        loc=segs[3]
        continue
    elif segs[2]=='2' and segs[0]==uid:
        print('%s\t%s'%(segs[3],loc))
    elif segs[2]=='2':
        #users中没有关联的uid，过滤掉。在
        #SQL中，此时右边表没有对应的元组，
        #记为NULL。
        pass
    #print('%s\t%s'%(segs[3],loc))

第二阶段：

map2.py

恒等映射

#!/usr/bin/python
#coding=utf-8
import sys

    print('%s'%(line.strip()))

reduce2.py

取出每个产品，并对其对应的不同地址计数。用python中的dict和set数据结构完成计数。

#!/usr/bin/python
#coding=utf-8
import sys
kc={}
for line in sys.stdin:
    line=line.strip()
    #print(line)
    segs=line.split('\t')
    if segs[0] not in kc:
        kc[segs[0]]={segs[1]}
    else:
        kc[segs[0]].add(segs[1])

for k,v in kc.items():
    print('%s\t%d'%(k,len(v)))

提交脚本：将两个阶段，前后提交mr任务。

#!/bin/bash
#hadoop 2.7.6
JAR=${HADOOP_HOME}/share/hadoop/tools/lib/hadoop-streaming-2.7.6.jar
INPUT=/chap04/input/
OUTPUT=output
MAPPER=map.py
REDUCER=reduce.py
hadoop fs -rm -r  $OUTPUT
#进行连接
hadoop jar $JAR \
-D stream.map.input.ignoreKey=true \
-D stream.map.output.field.separator='\t' \
-D stream.num.map.output.key.fields=2 \
-D mapreduce.map.output.key.field.separator='\t' \
-D mapreduce.partition.keypartitioner.options=-k1,1 \
-D mapreduce.job.output.key.comparator.class=org.apache.hadoop.mapreduce.lib.partition.KeyFieldBasedComparator \
-D mapreduce.partition.keycomparator.options=-k1,2 \
-input $INPUT \
-output $OUTPUT \
-mapper $MAPPER \
-reducer $REDUCER \
-file $MAPPER \
-file $REDUCER \
-partitioner org.apache.hadoop.mapred.lib.KeyFieldBasedPartitioner 

#统计每个产品的不同地址数
INPUT=output
OUTPUT=output2
MAPPER=map2.py
REDUCER=reduce2.py
hadoop fs -rm -r  $OUTPUT
hadoop jar $JAR \
-D stream.map.input.ignoreKey=true \
-D stream.map.output.field.separator='\t' \
-D stream.num.map.output.key.fields=1 \
-input $INPUT \
-output $OUTPUT \
-mapper $MAPPER \
-reducer $REDUCER \
-file $MAPPER \
-file $REDUCER \
-partitioner org.apache.hadoop.mapred.lib.KeyFieldBasedPartitioner

用数据库SQL验证MapReduce的结果

构造数据库表的SQL：

DROP TABLE if EXISTS users;
CREATE TABLE users(
	uid	VARCHAR(100),
	locid VARCHAR(100)
);

INSERT INTO users VALUES
('u1','UT'),
('u2','GA'),
('u3','CA'),
('u4','CA'),
('u5','GA'),
('u7',NULL);

DROP TABLE if EXISTS trans;
CREATE TABLE trans(
	tid VARCHAR(100),
	pid VARCHAR(100),
	uid VARCHAR(100),
	qty VARCHAR(100),
	amt VARCHAR(100)
);
INSERT INTO trans VALUES
('t1','p3','u1','3','330'),
('t2','p1','u2','1','400'),
('t3','p1','u1','3','600'),
('t4','p2','u2','10','1000'),
('t5','p4','u4','9','90'),
('t6','p1','u1','4','120'),
('t7','p4','u1','8','160'),
('t8','p4','u5','2','40'),
('t9','p4','u6','3','50'),
('t10','p3','u7','4','60');

SQL查询：

SELECT pid,count(distinct locid)
FROM trans AS t LEFT JOIN users AS u ON (t.uid = u.uid)
GROUP BY pid

查询结果：

+------+-----------------------+
| pid  | count(distinct locid) |
+------+-----------------------+
| p1   |                     2 |
| p2   |                     1 |
| p3   |                     1 |
| p4   |                     3 |
+------+-----------------------+

SQL查询的结果与MR任务的结果相同，证明其正确性。
